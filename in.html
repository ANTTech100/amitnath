Multi-Tenancy & Data Isolation

🏢
Organization Isolation

Each organization (tenant) is completely isolated by a unique token or organizational identifier. This ensures that data remains secure and private between different organizations using the platform.

👥
Role-Based Access Control

Administrators can only see and manage data within their own organization, including users, templates, questions, and responses. Users interact only with content specific to their organization.

🔧
Superadmin Privileges

The Superadmin role has system-wide access and can view and manage all data across all organizations. This role is essential for platform maintenance, support, and global analytics.
🔒 Security & Privacy
All API endpoints automatically filter data by tenant unless the ?all=true parameter is used by superadmin. This ensures complete data isolation and prevents accidental cross-tenant data exposure.

API Endpoints

Superadmin Endpoints

Endpoint Description
/api/users?all=true Get all users across all organizations (superadmin only)
/api/admins?all=true Get all administrators across all organizations (superadmin only)
/api/templates?all=true Get all templates across all organizations (superadmin only)
/api/questions?all=true Get all questions across all organizations (superadmin only)
/api/responses?all=true Get all responses across all organizations (superadmin only)
